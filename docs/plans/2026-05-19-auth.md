# Implementation Plan: Auth (BFF-only session cookies)

Design: `docs/designs/2026-05-19-auth.md`

Team lead spawns 3–5 team members. Each phase unlocks when the previous one finishes. Within a phase, tasks share no files and run in parallel. Backend team members follow the `vertical-slice-architecture`, `validation`, `error-handling`, `id-generation`, and `component-testing` skills. Frontend team members read `frontend/AGENTS.md` and `frontend/node_modules/next/dist/docs/` before writing any code.

**Files:** `backend/src/AiIssueTracker.Api/Common/Exceptions/DomainException.cs`, `backend/src/AiIssueTracker.Api/Common/Exceptions/GlobalExceptionHandler.cs`

### Task 2.4: Validation pipeline behavior

### Task 2.5: BFF auth primitives
- [x] `BffAuthOptions` (section `BffAuth`) with `SharedSecret`

- [x] `BffAuthenticationHandler : AuthenticationHandler<BffAuthenticationOptions>` — accepts requests with `X-Bff-Secret` matching the configured value; if `X-User-Id` is present, decode via `IdEncoding.TryDecode` and add `ClaimTypes.NameIdentifier` (raw `long.ToString()`); on success return success ticket
- [x] `ICurrentUser` / `CurrentUser` — reads `HttpContextAccessor.HttpContext.User`, exposes `Id` (`long?`), `IsAuthenticated`
- [x] `IPasswordHashing` / `PasswordHashing` — wraps `IPasswordHasher<User>`, exposes `Hash(User, password)` and `Verify(User, hash, password)` returning `bool`

### Task 2.6: Data layer + initial migration

- [x] `Data/AppDbContext.cs` — `DbSet<User> Users`, configure unique index on `Email`, `PasswordHash` required, max lengths matching the design (email 254, name 100)

### Task 2.7: Frontend BFF route handlers

## Phase 3 — Auth slices + composition

*Depends on: Phase 2 (identity, auth primitives, validation, exceptions, data layer). 4 parallel tasks — each owns its own file; `Program.cs` is a single task to avoid conflicts.*

### Task 3.1: Register slice
- [x] Static `Register` class with nested `Endpoint`, `Body`, `Request`, `Response`, `RequestValidator`, `RequestHandler`
- [x] `POST /api/auth/register` requires `AuthPolicies.BffOnly`
- [x] Validator: email non-empty / valid / ≤254 chars; password ≥8 / ≤128; name non-empty / ≤100. Error codes per `auth:user:{field}:{rule}` scheme
- [x] Handler: lower-case email, conflict-check via `AnyAsync`, throw `ConflictException("auth:user:email:already_exists")` if taken, create User with `IdFactory.Create()`, hash via `IPasswordHashing.Hash`, persist, return `Response(IdEncoding.Encode(id), email, name)`

**Files:** `backend/src/AiIssueTracker.Api/Features/Auth/Register.cs`

### Task 3.2: Login slice
- [x] Same vertical-slice shape, `POST /api/auth/login`, `BffOnly` policy
- [x] Validator: email + password required (no length/format rules — that's for register)
- [x] Handler: lookup by normalized email; if missing or `Verify` fails throw `UnauthorizedDomainException("auth:credentials:invalid")` (do not leak which one); on success return `Response(encodedId, email, name)`

**Files:** `backend/src/AiIssueTracker.Api/Features/Auth/Login.cs`

### Task 3.3: Me slice
- [x] `GET /api/auth/me`, requires `AuthPolicies.RequireUser`
- [x] Handler: read `ICurrentUser.Id`, look up the user, throw `NotFoundException("auth:user:not_found")` if absent (stale BFF session), return `Response(encodedId, email, name)`

**Files:** `backend/src/AiIssueTracker.Api/Features/Auth/Me.cs`

### Task 3.4: Program.cs composition
- [x] `AddServiceDefaults`, `AddNpgsqlDbContext<AppDbContext>("issuetracker")`
- [x] `AddIdFactory(...)`, `AddHttpContextAccessor`, register `ICurrentUser`, `IPasswordHashing`, `PasswordHasher<User>`
- [x] `Configure<BffAuthOptions>(BffAuth section)`
- [x] `AddAuthentication(BffAuthenticationHandler.SchemeName).AddScheme<...>(...)`; `AddAuthorization` with `BffOnly` and `RequireUser` policies
- [x] `AddMediatR(...)`, register `ValidationBehavior<,>` as open-generic `IPipelineBehavior<,>`, `AddValidatorsFromAssembly(..., includeInternalTypes: true)`
- [x] `AddEndpoints(Assembly.GetExecutingAssembly())`, `AddExceptionHandler<GlobalExceptionHandler>`, `AddProblemDetails()`
- [x] On startup, scope-resolve `AppDbContext` and `Database.MigrateAsync()`
- [x] Pipeline: `UseExceptionHandler()`, `UseAuthentication()`, `UseAuthorization()`, `MapDefaultEndpoints()`, `MapEndpoints()`, `Run()`
- [x] Add `public partial class Program;` sentinel for the test fixture

**Files:** `backend/src/AiIssueTracker.Api/Program.cs`

## Phase 5 — Auth slice tests

*Depends on: Phase 4. 3 parallel tasks, one per slice. Each file owns its own `ValidatorTests` nested class.*

### Task 5.1: Register tests
- [x] Happy path: BFF client posts a valid body → 200 with `{id, email, name}`, email lower-cased, user row exists, `PasswordHash` not equal to plain password
- [x] Duplicate email → 409 ProblemDetails with `errorCode: auth:user:email:already_exists`
- [x] Missing BFF secret → 401
- [x] Nested `ValidatorTests` covering: required/format/length rules for email, length rules for password, required/length for name — using `FluentValidation.TestHelper` and checking `errorCode`

**Files:** `backend/tests/AiIssueTracker.Api.Tests/Features/Auth/RegisterTests.cs`

### Task 5.2: Login tests
- [x] Happy path: seed user via DatabaseHarness with a hashed password → 200 with the matching DTO
- [x] Wrong password → 401 ProblemDetails with `errorCode: auth:credentials:invalid`
- [x] Unknown email → same 401 + same `errorCode` (no user enumeration)
- [x] Missing BFF secret → 401
- [x] Nested `ValidatorTests` covering: email required, password required

**Files:** `backend/tests/AiIssueTracker.Api.Tests/Features/Auth/LoginTests.cs`

### Task 5.3: Me tests
- [x] Happy path: seed user; user-scoped HTTP client (`X-Bff-Secret` + `X-User-Id`) → 200 with the seeded user's DTO
- [x] Missing `X-User-Id` (BFF-only client) → 401 / 403 per policy semantics
- [x] Stale `X-User-Id` (user deleted) → 404 ProblemDetails with `errorCode: auth:user:not_found`
- [x] Missing BFF secret → 401
- [x] No `RequestValidator` on this slice, so no nested `ValidatorTests`

**Files:** `backend/tests/AiIssueTracker.Api.Tests/Features/Auth/MeTests.cs`
