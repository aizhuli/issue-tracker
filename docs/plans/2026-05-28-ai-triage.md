# Implementation Plan: AI Triage (single-shot priority / labels / acceptance-criteria suggestion)

Design: `docs/designs/2026-05-28-ai-triage.md`
Depends on: `docs/plans/2026-05-26-issues.md` (Issue/Label entities, `IssueFull`/`LabelSummary` DTOs, `IssueMapping`, `<IssueDetail>` edit form, BFF passthrough plumbing, test harness — all in `main`). Also relies on the auth slice (`BffAuth`, `RequireUser`, `ICurrentUser`, `IdEncoding`, `DomainException` family, `ValidationBehavior`, `GlobalExceptionHandler`).

## Context

- **Packages already present** in `AiIssueTracker.Api.csproj`: `Microsoft.Extensions.AI` 10.6.0, `Microsoft.Extensions.AI.OpenAI` 10.6.0, `OpenAI` 2.10.0. No package work needed. The test project transitively references the abstractions.
- **Endpoints, MediatR requests, and validators auto-register** by assembly scan (`AddEndpoints` / `AddMediatR` / `AddValidatorsFromAssembly` in `Program.cs`). Creating the slice file is enough — no manual wiring.
- `GlobalExceptionHandler.BuildDomainProblem` already renders **any** `DomainException` using its `StatusCode`. A 502 only needs a new `BadGatewayException` subclass plus a `502 => "Bad Gateway"` entry in the handler's local `ReasonPhrases` switch.
- The triage slice returns `LabelSummary` (from `Features/Issues/IssueDtos.cs`) — it only **reads** that type, never edits the file.
- Backend feature dir `Features/Ai/` exists and is empty. Reuse the `Features/Issues/` slice patterns. Backend members follow `vertical-slice-architecture`, `validation`, `error-handling`, and `component-testing`. Frontend members follow `ui-design` and consult `frontend/node_modules/next/dist/docs/` before any Next.js 16 code (per `frontend/AGENTS.md`).
- **The `AcceptanceCriteriaAiSuggested` flag is being removed** (the design dropped it as redundant). Backend references: `Data/Entities/Issue.cs`, `Features/Issues/IssueDtos.cs`, `CreateIssue.cs`, `GetIssue.cs`, `UpdateIssue.cs`, `ChangeStatus.cs`, `ChangeAssignee.cs`, the migration snapshot, and `Features/Issues/CreateIssueTests.cs`. Frontend references: `lib/types/issues.ts`, `components/issues/IssueDetail.tsx`, and the `IssueDetail`/`CreateIssueModal` test files.
- **Cross-phase note (frontend):** Phase 1 removes `acceptanceCriteriaAiSuggested` from the TS type; its consumers (`IssueDetail.tsx` + two test files) are fixed in Phase 2. The frontend will not fully type-check between the two phases — this is expected; Phase 2 closes it.
- Recommended team size: 4 members.

- [ ] Create `Integrations/Llm/LlmOptions.cs` — `{ string? BaseUrl, string? Model, string? ApiKey, int TimeoutSeconds = 30 }`, `const string SectionName = "Llm"`.
- [ ] Create `Integrations/Llm/LlmRegistration.cs` — an extension `AddLlmChatClient(this IHostApplicationBuilder builder)` that binds `LlmOptions` and registers `IChatClient` through a **lazy factory** so a missing endpoint at startup does not throw (only fails when actually resolved without config): `new OpenAIClient(new ApiKeyCredential(opts.ApiKey ?? (Ollama is OpenAI-compatible at `/v1`, so the same registration serves local and cloud.)
- [ ] In `Program.cs`, add `builder.Services.Configure<LlmOptions>(builder.Configuration.GetSection(LlmOptions.SectionName));` and `builder.AddLlmChatClient();` near the other service registrations.
- [ ] In `AppHost.cs`, add an `llm-base-url` parameter (+ `llm-model`, and a secret `llm-api-key`), and pass them to the API via `.WithEnvironment("Llm__BaseUrl", …)`, `Llm__Model`, `Llm__ApiKey`.
- [ ] In `appsettings.json` / `appsettings.Development.json`, add a placeholder `"Llm": { "BaseUrl": "", "Model": "", "ApiKey": "" }` with a comment that `ApiKey` should come from user-secrets, not source control.
- [ ] `dotnet build backend/src/AiIssueTracker.Api` — compiles.

### Task 1.2: Triage vertical slice + 502 exception

- [ ] Create `Features/Ai/TriageIssue.cs` following the slice shape (`static class` with `Endpoint : IEndpoint`, `Request`, `RequestValidator`, `RequestHandler`).
  - **Endpoint:** `MapPost("/api/projects/{slug}/issues/{number}/ai/triage", …)` with `[FromBody] Body(string Title, string? Description)`, `.RequireAuthorization(AuthPolicies.RequireUser)`, `.WithTags("Ai")`, `.WithName("TriageIssue")`. Returns `Results.Ok(TriageSuggestion)`.
  - **Response record** `TriageSuggestion(string Priority, LabelSummary[] Labels, string AcceptanceCriteria)` (reuse `LabelSummary` from `Features.Issues` via `using`).
  - **Request** `(string Slug, int Number, string Title, string? Description) : IRequest<TriageSuggestion>`.
  - **Validator:** `Title` `NotEmpty().MaximumLength(200)` → `ai:triage:title:required_or_too_long`; `Description` `MaximumLength(10000).When(x => x.Description is not null)` → `ai:triage:description:too_long`.
- [ ] **Handler** (inject `AppDbContext`, `IChatClient`): resolve project by slug → 404 `projects:project:not_found`; verify issue exists by `(ProjectId, Number)` → 404 `issues:issue:not_found`; load the project's labels (id/name/color).
- [ ] Build the prompt **inside the slice**: a system message (role, the four priorities, "choose labels ONLY from this list; empty array if none fit; never invent labels", strict-JSON output) + a user message carrying project name + description, the label names, and the **body** title/description. Call `IChatClient.GetResponseAsync` with `ChatResponseFormat.Json` (when supported), low temperature (~0.2), and a `CancellationToken` bounded by `LlmOptions.TimeoutSeconds`.
- [ ] Wrap the chat call in try/catch → `BadGatewayException` `ai:triage:llm:unavailable`. Deserialize into an internal `{ priority, labels[], acceptanceCriteria }`; on parse failure retry once, then throw `BadGatewayException` `ai:triage:llm:invalid_response`.
- [ ] Resolve defensively: **priority** via `IssueMapping.IsValidPriority` else fall back to the issue's current `Priority.ToKebab()`; **labels** case-insensitive match against project labels → `LabelSummary[]`, drop misses; **acceptanceCriteria** trim + cap at 10 000 chars. Return `TriageSuggestion`. **No DB writes.**
- [ ] Add `BadGatewayException(string message, string errorCode) : DomainException(message, errorCode, StatusCodes.Status502BadGateway)` to `Common/Exceptions/DomainException.cs`.
- [ ] Add `502 => "Bad Gateway"` to the `ReasonPhrases.For` switch in `Common/Exceptions/GlobalExceptionHandler.cs`.
- [ ] `dotnet build` — compiles.

**Files:** `backend/src/AiIssueTracker.Api/Features/Ai/TriageIssue.cs`, `backend/src/AiIssueTracker.Api/Common/Exceptions/DomainException.cs`, `backend/src/AiIssueTracker.Api/Common/Exceptions/GlobalExceptionHandler.cs`
