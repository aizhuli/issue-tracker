## Phase 2 — Core auth components

*Depends on: Phase 1 (KanbanBoard + mock data from 1.3; UI primitives from 1.2; schemas from 1.1).*

*Tasks 2.1 and 2.2 touch different files — run in parallel.*

### Task 2.1: Demo board animation + DemoBoard component

- [x] Create `frontend/components/auth/useDemoAnimation.ts`
  - [x] Accept `(issues: MockIssue[], setIssues: Dispatch<SetStateAction<MockIssue[]>>)` as arguments
  - [x] Timer 1 (3.5s): pick a random non-`done` issue, advance its status one step using functional `setIssues` update; if it just reached `done`, schedule a reset to `backlog` after 2s via `setTimeout`
  - [x] Timer 2 (6s): insert a new mock issue (`createMockIssue`, `entering: true`) into a random non-`done` column; clear `entering` flag after 300ms via `setTimeout`; after 4 new cards accumulate, pause the timer for 12s then resume
  - [x] Interval IDs stored as local constants inside `useEffect`; cleared in the cleanup return. `useRef` used for new-card count and pause flag.
- [x] Create `frontend/components/auth/DemoBoard.tsx`
  - [x] Initialize state from `MOCK_ISSUES`; pass state + setter to `useDemoAnimation`
  - [x] Render `KanbanBoard` with `position: absolute; inset: 0`, `pointerEvents: 'none'`, `userSelect: 'none'`, `filter: 'blur(5px) saturate(0.85) brightness(0.97)'`, `transform: 'scale(1.06)'`
  - [x] No right-edge gradient overlay — board is blurred full-screen behind the centered panel

**Files:** `frontend/components/auth/useDemoAnimation.ts`, `frontend/components/auth/DemoBoard.tsx`

### Task 2.2: Login and Register forms

- [x] Create `frontend/components/auth/LoginForm.tsx`
  - [x] Uses `useState` for field values and error state (no `react-hook-form`)
  - [x] Render `FormField` for email, `PasswordInput` for password, `FormError` for form-level errors
  - [x] On submit: set loading state → `POST /api/auth/login` with `{ email, password }` → on success read `returnTo` from `useSearchParams`, redirect via `router.push`
  - [x] Map `errorCode` values from the response to field errors or form-level error per the error table in the design doc
  - [x] CTA button: full width, `height: 40px`, `background: var(--accent-1)`, `color: var(--accent-1-ink)`, `border-radius: 9px`; shows a CSS `border`-based spinner and disables while loading
- [x] Create `frontend/components/auth/RegisterForm.tsx`
  - [x] Same pattern as `LoginForm` with `registerSchema`; adds a `FormField` for name above email
  - [x] `POST /api/auth/register`; on success redirect via `router.push` (same `returnTo` logic)
  - [x] Map all register-specific `errorCode` values per the design doc

**Files:** `frontend/components/auth/LoginForm.tsx`, `frontend/components/auth/RegisterForm.tsx`
