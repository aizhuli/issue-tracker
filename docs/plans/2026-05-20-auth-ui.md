# Implementation Plan: Auth UI

Design: `docs/designs/2026-05-20-auth-ui.md`
Backend auth design: `docs/designs/2026-05-19-auth.md`

## Context

BFF route handlers and `lib/session.ts` / `lib/api-client.ts` already exist from the backend auth slice.
The `design_files/` directory holds the Aigo design system — reference `styles.css`, `board.jsx`, `data.js`, `chrome.jsx`, and `drawer.jsx` when porting.

---

## Phase 1 — Foundation

*No dependencies. All four tasks run in parallel.*

### Task 1.1: CSS design tokens + Zod schemas

- [ ] Open `design_files/styles.css` and identify all CSS custom properties (surfaces, ink, accent, status colours, radii, shadows)
- [ ] Merge those tokens into `frontend/app/globals.css` under `:root`, preserving any existing Next.js resets
- [ ] Add the `slide-in` keyframe (`translateX(20px) → 0`, 220ms) and `.slide-in` utility class to `globals.css`
- [ ] Create `frontend/lib/schemas/auth.ts` with `loginSchema` and `registerSchema` exactly as specified in the design doc
- [ ] Export both schemas and their inferred TypeScript types (`LoginFormValues`, `RegisterFormValues`)

**Files:** `frontend/app/globals.css`, `frontend/lib/schemas/auth.ts`

---

### Task 1.2: UI primitive components

- [ ] Create `frontend/components/ui/FormField.tsx` — renders a `<label>` + `<input>` pair; accepts `label`, `error?: string`, `inputProps`; when `error` is set, shifts input border to `#B94D2F` and renders the message below in `color: #B94D2F`, `font-size: 12px`
- [ ] Create `frontend/components/ui/PasswordInput.tsx` — wraps `FormField` (or uses a raw input); adds a show/hide toggle button with an eye icon (inline SVG or the `Icon` component if ported); toggles `type="password"` ↔ `type="text"`
- [ ] Create `frontend/components/ui/FormError.tsx` — pill component; `background: #F0DDDD`, `color: #7A3535`, `border-radius: 8px`, `padding: 8px 12px`, `font-size: 12.5px`; renders only when `message` prop is truthy

**Files:** `frontend/components/ui/FormField.tsx`, `frontend/components/ui/PasswordInput.tsx`, `frontend/components/ui/FormError.tsx`

---

### Task 1.3: KanbanBoard port + mock data

- [ ] Read `design_files/data.js` and create `frontend/lib/mock-issues.ts` — export a typed `Issue` interface (`id`, `title`, `status`, `priority`, `label?`) and the `MOCK_ISSUES` array (≥10 items spread across all five statuses)
- [ ] Read `design_files/board.jsx` and create `frontend/components/kanban/KanbanBoard.tsx` — TypeScript port of the kanban component; accepts `issues: Issue[]` prop; renders five columns (backlog, todo, in-progress, in-review, done) using design tokens from CSS vars; apply existing card CSS transitions so cards animate naturally when `issues` prop changes
- [ ] Ensure the component is purely presentational — no internal state, no drag-and-drop logic

**Files:** `frontend/lib/mock-issues.ts`, `frontend/components/kanban/KanbanBoard.tsx`

---

### Task 1.4: Next.js middleware

- [ ] Create `frontend/middleware.ts`
- [ ] Import `getIronSession` and the session config from `lib/session.ts`
- [ ] Define the matcher: all routes except `/login`, `/register`, `/api/auth/*`, and Next.js internals (`/_next/*`, `/favicon.ico`)
- [ ] In the middleware handler, read the session; if no `userId`, redirect to `/login?returnTo=<current path>`
- [ ] Redirect authenticated users away from `/login` and `/register` to `/`

**Files:** `frontend/middleware.ts`

## Phase 2 — Core auth components

*Depends on: Phase 1 (KanbanBoard + mock data from 1.3; UI primitives from 1.2; schemas from 1.1).*

*Tasks 2.1 and 2.2 touch different files — run in parallel.*

### Task 2.1: Demo board animation + DemoBoard component

- [ ] Create `frontend/components/auth/useDemoAnimation.ts`
  - [ ] Accept `(issues: Issue[], setIssues: Dispatch<SetStateAction<Issue[]>>)` as arguments
  - [ ] Timer 1 (3.5s): pick a random non-`done` issue, advance its status one step; if it just reached `done`, schedule a reset to `backlog` after 2s via `setTimeout`
  - [ ] Timer 2 (6s): insert a new mock issue into a random non-`done` column; after 4 new cards accumulate, pause the timer for 12s then resume
  - [ ] New cards enter via a CSS class toggled one frame after mount using `requestAnimationFrame` (`opacity: 0 → 1`, `transform: translateY(-6px) → 0`, 250ms)
  - [ ] Store both interval IDs in `useRef`; clear them in the `useEffect` cleanup
- [ ] Create `frontend/components/auth/DemoBoard.tsx`
  - [ ] Initialize state from `MOCK_ISSUES`; pass state + setter to `useDemoAnimation`
  - [ ] Render `KanbanBoard` wrapped in a `div` with `pointerEvents: 'none'`, `userSelect: 'none'`, `filter: 'saturate(0.85)'`, and `flex: 1; overflow: hidden`
  - [ ] Render a `position: absolute` overlay div at the right edge of the board: `width: 60px`, `background: linear-gradient(to right, transparent 0%, var(--paper) 100%)`

**Files:** `frontend/components/auth/useDemoAnimation.ts`, `frontend/components/auth/DemoBoard.tsx`

---

### Task 2.2: Login and Register forms

- [ ] Create `frontend/components/auth/LoginForm.tsx`
  - [ ] Use `react-hook-form` with `zodResolver(loginSchema)` (or manual state — check if `react-hook-form` is installed; if not, use `useState`)
  - [ ] Render `FormField` for email, `PasswordInput` for password, `FormError` for form-level errors
  - [ ] On submit: set loading state → `POST /api/auth/login` with `{ email, password }` → on success read `returnTo` from `useSearchParams`, redirect via `router.push`
  - [ ] Map `errorCode` values from the response to field errors or form-level error per the error table in the design doc
  - [ ] CTA button: full width, `height: 40px`, `background: var(--accent-1)`, `color: var(--accent-1-ink)`, `border-radius: 9px`; shows a CSS `border`-based spinner and disables while loading
- [ ] Create `frontend/components/auth/RegisterForm.tsx`
  - [ ] Same pattern as `LoginForm` with `registerSchema`; adds a `FormField` for name above email
  - [ ] `POST /api/auth/register`; on success redirect via `router.push` (same `returnTo` logic)
  - [ ] Map all register-specific `errorCode` values per the design doc

**Files:** `frontend/components/auth/LoginForm.tsx`, `frontend/components/auth/RegisterForm.tsx`

## Phase 3 — Shell and pages

*Depends on: Phase 2 (AuthPanel needs LoginForm/RegisterForm from 2.2; AuthShell needs DemoBoard from 2.1).*

### Task 3.1: AuthPanel, AuthShell, layout, and pages

- [ ] Create `frontend/components/auth/AuthPanel.tsx`
  - [ ] Read `design_files/chrome.jsx` to copy the app logo SVG; render it + "Aigo - Issue Board" as the panel header
  - [ ] Render a tab strip with `.dr-tabs` / `.dr-tab` classes (reference `design_files/drawer.jsx`); active tab has `borderBottom: '2px solid var(--ink-0)'`
  - [ ] Accept `activeTab: 'login' | 'register'` prop; clicking a tab calls `router.push('/login')` or `router.push('/register')`
  - [ ] Render `LoginForm` when `activeTab === 'login'`, `RegisterForm` when `activeTab === 'register'`
  - [ ] Pin a footer at the panel bottom: `color: var(--ink-3)`, `fontSize: 11px`, "© 2025 Aigo"
- [ ] Create `frontend/components/auth/AuthShell.tsx`
  - [ ] Full-viewport layout (`width: 100vw; height: 100vh; display: flex; position: relative; overflow: hidden; background: var(--paper)`)
  - [ ] Left zone: renders `DemoBoard` with `flex: 1; height: 100%`
  - [ ] Right panel: `width: 380px`, `height: 100%`, `background: var(--paper)`, `borderLeft: 1px solid var(--border)`, `boxShadow: var(--shadow-lg)`, `overflowY: auto`; wrapped in a `div` with `className="slide-in"` so it animates on mount
  - [ ] Brand pill: `position: absolute; top: 16px; left: 18px`; renders logo SVG + "Aigo - Issue Board" in `.sb-ws-name` style
  - [ ] Forwards `activeTab` prop to `AuthPanel`
- [ ] Create `frontend/app/(auth)/layout.tsx` — `'use client'`; renders `<AuthShell activeTab={...}>` passing the active tab derived from the pathname; no sidebar, topbar, or other chrome
- [ ] Create `frontend/app/(auth)/login/page.tsx` — thin page; passes `activeTab="login"` to the layout (or renders `AuthShell` directly with that prop)
- [ ] Create `frontend/app/(auth)/register/page.tsx` — same with `activeTab="register"`

**Files:** `frontend/components/auth/AuthPanel.tsx`, `frontend/components/auth/AuthShell.tsx`, `frontend/app/(auth)/layout.tsx`, `frontend/app/(auth)/login/page.tsx`, `frontend/app/(auth)/register/page.tsx`
