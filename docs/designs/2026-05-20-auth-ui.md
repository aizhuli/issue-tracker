# Auth UI Design

Date: 2026-05-20
Scope: Next.js frontend auth pages (`/login`, `/register`). No backend changes.

## Overview

The auth pages render a full-viewport animated demo board behind a right-anchored side panel. Users see the product working before they log in. The panel contains a tabbed form for Log in / Sign up.

---

## Page Shell & Layout

Both `/login` and `/register` render the same shell component (`AuthShell`). There is no sidebar, topbar, or viewbar ‚Äî those are post-login chrome.

**Left zone (flexible width):** `DemoBoard` fills all space to the left of the panel. It renders `KanbanBoard` fed `MOCK_DATA`, wrapped in `pointer-events: none` + `user-select: none`. The board runs at `filter: saturate(0.85)` so its accent colours don't compete with the form. A `linear-gradient(to right, transparent 0%, var(--paper) 100%)` gradient over the last `60px` of the right edge fades the board into the panel border.

**Right panel:** `380px` wide, fixed to the right edge, full height. Styled as `.drawer`: `background: var(--paper)`, `border-left: 1px solid var(--border)`, `box-shadow: var(--shadow-lg)`. Slides in from the right on mount using the existing `slide-in` keyframe (`translateX(20px) ‚Üí 0`, 220ms).

**Brand mark:** A small floating pill ‚Äî `position: absolute; top: 16px; left: 18px` ‚Äî over the board. Renders the app logo SVG (from `chrome.jsx`) + the text **"Aigo - Issue Board"** in `.sb-ws-name` style. Grounds the brand without a full sidebar.

**Route protection:** Next.js middleware redirects authenticated users away from `/login` and `/register` immediately.

## Auth Panel Internals

### Header

App logo SVG + "Aigo - Issue Board" in `.sb-ws-name` style at the top of the panel.

### Tab strip

Two tabs using `.dr-tabs` / `.dr-tab` classes:
- **Log in** (`/login`)
- **Sign up** (`/register`)

Active tab: `border-bottom: 2px solid var(--ink-0)`. Switching tabs calls `router.push('/login')` or `router.push('/register')` ‚Äî no full reload.

### Log in form fields

1. **Email** ‚Äî labeled input, full width
2. **Password** ‚Äî labeled input + show/hide toggle (eye icon via `Icon` component), full width
3. **CTA:** "Log in" button ‚Äî full width, `height: 40px`, `background: var(--accent-1)`, `color: var(--accent-1-ink)`, `border-radius: 9px`

### Sign up form fields

1. **Name** ‚Äî labeled input, full width
2. **Email** ‚Äî labeled input, full width
3. **Password** ‚Äî labeled input + show/hide toggle, full width
4. **CTA:** "Create account" button ‚Äî same style as Log in CTA

### Inline field errors

Each field shows a small `color: #B94D2F` message below it when validation fails. The field border shifts to `#B94D2F`. Errors are mapped from BFF `errorCode` values (see Error Handling section).

### Form-level errors

Rendered as a pill above the CTA: `background: #F0DDDD`, `color: #7A3535`, `border-radius: 8px`, `padding: 8px 12px`, `font-size: 12.5px`. Used for `auth:credentials:invalid` and network/5xx failures.

### Loading state

CTA disables and shows a CSS `border`-based spinner (no library) while the BFF call is in flight.

### Panel footer

`color: var(--ink-3)`, `font-size: 11px` ‚Äî "¬© 2025 Aigo" ‚Äî pinned to the bottom of the panel.

## Board Animation

Implemented as a single hook `useDemoAnimation(issues, setIssues)`. Two independent `setInterval` timers start on mount and are cleared on unmount. Both timers use `useRef` to hold the interval ID, avoiding stale closures.

### Timer 1 ‚Äî card move (every 3.5s)

Picks a random issue that is not in `done`, advances it one status step:
```
backlog ‚Üí todo ‚Üí in-progress ‚Üí in-review ‚Üí done
```
Issues that reach `done` reset to `backlog` after a 2s pause so the board never empties. Applied via `setIssues` ‚Äî `KanbanBoard` re-renders naturally with its existing CSS transitions.

### Timer 2 ‚Äî card appear (every 6s)

Inserts a new mock issue into a random non-`done` column. The card enters with `opacity: 0 ‚Üí 1` + `transform: translateY(-6px) ‚Üí 0` over 250ms (CSS class toggled one frame after mount via `requestAnimationFrame`). After 4 new cards accumulate the timer pauses for 12s before resuming, preventing board overflow.

## Error Handling & Redirect Flow

### Submission flow

1. Client-side Zod validation runs first. Errors appear instantly without a network call.
2. If Zod passes, the form `POST`s to the BFF route handler (`/api/auth/login` or `/api/auth/register`).
3. The BFF forwards to the ASP.NET API and passes ProblemDetails through unchanged.
4. The frontend reads `errorCode` from the response body and maps it to field-level or form-level messages.

### Error mapping

| `errorCode` | Where shown | Message |
|---|---|---|
| `auth:user:email:required` / `invalid_format` / `too_long` | Below email field | "Enter a valid email address" |
| `auth:user:password:required` / `too_short` / `too_long` | Below password field | "Password must be 8‚Äì128 characters" |
| `auth:user:name:required` / `too_long` | Below name field | "Name is required (max 100 chars)" |
| `auth:user:email:already_exists` | Below email field | "An account with this email already exists" |
| `auth:credentials:invalid` | Form-level (above CTA) | "Incorrect email or password" |
| Network / 5xx | Form-level | "Something went wrong ‚Äî please try again" |

### Redirect flow

- On success the BFF writes the session cookie and returns `{ userId, email, name }`.
- The frontend reads `returnTo` from the query string. If present and starts with `/`, redirects there. Otherwise redirects to `/`.
- Uses `router.push` ‚Äî no full page reload.

## Zod Schemas (`lib/schemas/auth.ts`)

```ts
export const loginSchema = z.object({
  email: z.string().email().max(254),
  password: z.string().min(1),
});

export const registerSchema = loginSchema.extend({
  name: z.string().min(1).max(100),
  password: z.string().min(8).max(128),
});
```

## Out of Scope

- Password reset, email verification, OAuth.
- Remember me / persistent sessions.
- CAPTCHA or rate-limit UI.
- Mobile / responsive layout (demo targets desktop).
- Vitest / Playwright tests for auth UI (deferred to a follow-up step).
