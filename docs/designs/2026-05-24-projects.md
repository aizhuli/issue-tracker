# Projects design — CRUD with owner-gated edit/delete

Date: 2026-05-24
Scope: Backend (vertical slice) + BFF passthrough + Frontend (list page + modals).
Depends on: 2026-05-19-auth (BffAuth scheme, `ICurrentUser`, foundations).

## Decisions

- **Permissions:** owner-only edit and delete. Anyone authenticated can list and
  view any project (matches "anyone can create issues, take work, comment" from
  CLAUDE.md). No project membership concept.
- **Slug:** user-provided, validated. FE auto-fills from name until the user
  touches the slug field. **Immutable after creation.**
- **Delete:** hard cascade. Removes the project and all its issues, labels,
  and comments. Slug becomes free again.
- **List visibility:** all projects, paginated, with optional `?q=` prefix search
  on name and slug.
- **Edit semantics:** PUT (full replacement) of `{ name, description }`. Slug
  read from URL identifies the resource; it is never rewritten.
- **Field limits:** name 1–100, description 0–2000, slug 3–50
  (`^[a-z0-9](?:[a-z0-9-]{1,48}[a-z0-9])?$`).
- **Slug collisions:** no auto-suffix. Backend returns 409 with
  `projects:project:slug:already_exists`; FE shows the error inline next to the
  slug field.

## Endpoints

All under `RequireUser` (valid `X-Bff-Secret` + `X-User-Id`).

| Method | Route                                       | Purpose                       |
|--------|---------------------------------------------|-------------------------------|
| POST   | `/api/projects`                             | Create                        |
| GET    | `/api/projects`                             | List (token-paginated, `?q=`) |
| GET    | `/api/projects/slug-availability?slug={s}`  | Live slug check               |
| GET    | `/api/projects/{slug}`                      | Get by slug                   |
| PUT    | `/api/projects/{slug}`                      | Edit (owner only)             |
| DELETE | `/api/projects/{slug}`                      | Delete (owner only, cascade)  |

### Contracts

**Create** — `POST /api/projects`

Request: `{ name: string (1–100), slug: string (3–50, regex), description?: string (0–2000) }`.
Response: `201 Created` with the full Project DTO, `Location: /api/projects/{slug}`.
Owner is set from `ICurrentUser.UserId`, never from the request.

**List** — `GET /api/projects?pageToken=&maxPageSize=&q=`

Follows the `token-pagination` skill (opaque encrypted token, `maxPageSize`
capped at 100). Default sort `created_at DESC, id DESC`. `q` is a
case-insensitive prefix match on name and slug.

Response item shape (`ProjectSummary`):

```json
{ "id": "01J...", "slug": "web-platform", "name": "Web Platform",
  "ownerId": "01J...", "ownerName": "Maya Chen", "createdAt": "..." }
```

**Get** — returns the full DTO including `description`, `updatedAt`,
`ownerId`, `ownerName`. Anyone authenticated can read.

**Edit (PUT)** — `{ name, description }`. The slug in the URL identifies the
project; it is not interpreted as a rename. Returns the updated DTO. 403 if
caller is not the owner.

**Delete** — `204 No Content`. Cascades via EF `OnDelete(Cascade)` on
`Issue.ProjectId`, `Label.ProjectId`, and (transitively) `Comment.IssueId`.
403 if not owner.

**Slug availability** — `GET /api/projects/slug-availability?slug={s}`

Response is always 200; it's a check, not a lookup:

```json
{ "slug": "web-platform", "available": false, "reason": "taken" }
```

`reason ∈ { "invalid_format", "taken" }` (omitted when `available: true`).
Cheap single indexed lookup. No rate-limiting in MVP.

## Data model

`Data/Entities/Project.cs`:

```csharp
public class Project
{
    public long Id { get; set; }                  // IdGen
    public string Slug { get; set; } = "";
    public string Name { get; set; } = "";
    public string? Description { get; set; }
    public long OwnerId { get; set; }
    public User Owner { get; set; } = null!;
    public DateTimeOffset CreatedAt { get; set; }
    public DateTimeOffset UpdatedAt { get; set; }

    public ICollection<Issue> Issues { get; set; } = [];
    public ICollection<Label> Labels { get; set; } = [];
}
```

EF configuration (in `AppDbContext.OnModelCreating`):

- `HasIndex(p => p.Slug).IsUnique()`. Slug always stored lowercase; comparison
  ordinal. No `citext` extension needed.
- `Property(p => p.Slug).HasMaxLength(50)`, `Name` 100, `Description` 2000.
- `HasOne(p => p.Owner).WithMany().HasForeignKey(p => p.OwnerId)
   .OnDelete(DeleteBehavior.Restrict)` — can't delete a user that owns
  projects (MVP doesn't support user deletion anyway).
- `Issue.ProjectId`, `Label.ProjectId`, `Comment.IssueId` (added in later
  slices) configured with `OnDelete(Cascade)` so project delete drops
  everything beneath it.
- `CreatedAt` / `UpdatedAt` are set explicitly in handlers from
  `TimeProvider.GetUtcNow()`. No interceptors.

Migration: `AddProjects`.

## Validation & error codes

FluentValidation, in the slice file alongside the handler.

```csharp
// CreateProject
RuleFor(x => x.Name).NotEmpty().MaximumLength(100)
    .WithErrorCode("projects:project:name:required_or_too_long");

RuleFor(x => x.Slug).NotEmpty()
    .Matches("^[a-z0-9](?:[a-z0-9-]{1,48}[a-z0-9])?$")
    .WithErrorCode("projects:project:slug:invalid_format");

RuleFor(x => x.Description).MaximumLength(2000)
    .When(x => x.Description is not null)
    .WithErrorCode("projects:project:description:too_long");

// UpdateProject — same name/description rules, no slug field.
// SlugAvailability — slug required; invalid format becomes a 200 response
// with reason:"invalid_format", not a 400.
```

Domain-level errors raised from handlers via `DomainException`:

| Code                                       | When                          | HTTP |
|--------------------------------------------|-------------------------------|------|
| `projects:project:slug:already_exists`     | unique-violation on create    | 409  |
| `projects:project:not_found`               | get/edit/delete unknown slug  | 404  |
| `projects:project:edit:forbidden`          | edit by non-owner             | 403  |
| `projects:project:delete:forbidden`        | delete by non-owner           | 403  |

ProblemDetails (RFC 7807) carrying `errorCode` and `traceId` per the
`error-handling` skill.

## Frontend

Routes (one new page, two modals — no dedicated create/edit pages):

```
app/(app)/projects/page.tsx
app/api/projects/route.ts                     # GET list, POST create
app/api/projects/[slug]/route.ts              # GET, PUT, DELETE
app/api/projects/slug-availability/route.ts   # GET
```

All BFF route handlers are pure passthrough: attach `X-Bff-Secret` and
`X-User-Id` from the iron-session, forward to the API, return the response
unchanged (including ProblemDetails). No validation or reshaping.

### `/projects` page

Standard authenticated shell (sidebar + topbar + viewbar).

- **Viewbar:** `.tb-search` bound to `?q=` on the right of the title; a primary
  `.sb-quick-btn.primary` "+ New project" button that opens
  `<ProjectFormModal mode="create">`.
- **Body:** responsive 3-column grid of `<ProjectCard>`. Card uses
  `--surface`, `--border`, `--radius`, `--shadow-sm`; hover lifts to
  `--shadow-md`. Content:
  - Project name — Manrope 600 / 15px.
  - Slug as a JetBrains Mono chip (`--surface-3` bg, `--radius-sm`).
  - Description preview, 3-line clamp, `--ink-2`.
  - `<Avatar id={ownerId} size={20}>` + owner name at bottom-left.
  - `dots` icon menu at bottom-right with Edit / Delete — only rendered when
    `ownerId === me.id`.
- **Empty state:** centered `<Icon name="folder">` + "No projects yet — create
  your first" + the same "+ New project" CTA.
- **Pagination:** "Load more" button at the end, appends next page from the
  token cursor.

### `<ProjectFormModal mode="create" | "edit" project?={...}>`

Uses `.ai-modal` / `.ai-scrim` with `ai-pop` entrance, Esc closes, focus trap.

Fields (stacked):

1. **Name** — text input, 1–100, error shown inline.
2. **Slug** — text input with availability indicator (see below). In `edit`
   mode, read-only with a small "(cannot be changed)" hint.
3. **Description** — textarea, `min-height: 96px`, 0–2000.

CTA: "Create project" / "Save changes" — full-width accent-1 button (existing
`.dr-compose-send` style). Disabled while slug-check or submit is in flight.

**Slug availability** (create mode):

- Local Zod check runs on every keystroke.
- Auto-fill from name slugifies as the user types in `name`, until the user
  edits slug — then a `slugTouched` ref stops auto-fill.
- If local Zod passes, debounce 400ms and call
  `GET /api/projects/slug-availability` with an `AbortController` that cancels
  in-flight requests on subsequent keystrokes.
- UI states next to the field:
  - typing / debouncing → muted "Checking…" with a small spinner
  - local format invalid → red text with the format hint, no BE call
  - BE says available → green check `<Icon name="enter">` + "Available"
  - BE says taken → red text "Already taken — pick a different one"
- Submit is disabled until the BE check reports `available: true`.
- On submit, if the API returns `projects:project:slug:already_exists`, the
  modal re-flags the slug field as taken using the same component — no
  special-case error path.

### `<DeleteProjectDialog>`

Small modal: "Delete *Project name*? This permanently removes all its issues,
labels, and comments." Requires typing the exact project name to enable the
red Delete button. Esc closes. No undo.

### Zod schemas

`frontend/lib/schemas/projects.ts` mirrors backend rules exactly:

```ts
export const slugSchema = z.string()
  .regex(/^[a-z0-9](?:[a-z0-9-]{1,48}[a-z0-9])?$/,
    "Lowercase letters, digits, hyphens. 3–50 chars.");

export const projectCreateSchema = z.object({
  name: z.string().min(1).max(100),
  slug: slugSchema,
  description: z.string().max(2000).optional(),
});

export const projectUpdateSchema = z.object({
  name: z.string().min(1).max(100),
  description: z.string().max(2000).optional(),
});
```

The shared `errorCode → field` mapping util gains entries for the four new
project error codes.

## Testing

Per the `component-testing` skill — HTTP-level tests with a real Postgres
Testcontainer and the `WebApplicationFactory` harness. No handler-level unit
tests.

### Backend

| Slice              | Cases                                                                                                                                                                                                                |
|--------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `CreateProject`    | 201 with full DTO and `Location` · 409 on slug collision · 400 invalid slug format · 400 name length boundaries (0, 101) · 401 missing `X-User-Id`                                                                  |
| `GetProject`       | 200 by slug · 404 unknown slug · works for non-owner (read is open)                                                                                                                                                  |
| `ListProjects`     | Pagination round-trip first → second → empty · `?q=foo` prefix match on name and slug · stable `created_at DESC, id DESC` sort across pages                                                                          |
| `UpdateProject`    | 200 by owner, fields updated, `UpdatedAt` bumped · 403 non-owner (`projects:project:edit:forbidden`) · 404 unknown slug · 400 on validation failures · slug column unchanged                                          |
| `DeleteProject`    | 204 by owner · cascade verified (seed issue + label + comment, assert all gone) · 403 non-owner · 404 unknown slug                                                                                                   |
| `SlugAvailability` | 200 `{available:true}` for unused valid slug · 200 `{available:false, reason:"taken"}` for existing · 200 `{available:false, reason:"invalid_format"}` for bad format · always 200, never 400                       |

Harness conveniences used: `harness.AsUser(userId)` to set `X-User-Id`,
`harness.Db` for direct seeding, deterministic `IdFactory` sequence so ID
assertions are stable.

### Frontend (Vitest + RTL)

- `ProjectFormModal` — slug auto-fills until touched · debounced availability
  call fires once per pause · green check on available · red text on taken ·
  submit disabled until check resolves available · 409 on submit re-flags
  field as taken using the same UI path.
- `DeleteProjectDialog` — Delete button stays disabled until project name is
  typed exactly · Esc closes.
- `lib/schemas/projects.ts` — boundary values (1, 100, 0, 2000, 3, 50) plus
  regex edges.

### E2E (optional, Playwright)

One smoke flow: log in → create project → see it in the list → edit
description → delete with confirm.

## Out of scope

- Project transfer / change-owner.
- Soft delete or recoverable trash.
- Project membership, per-user roles, or non-owner edit/delete.
- Slug rename + redirect from the old slug.
- Rate-limiting the slug-availability endpoint.
- Server-side full-text search (`?q=` is prefix-only for MVP).
