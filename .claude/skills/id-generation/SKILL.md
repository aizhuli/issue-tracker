# ID Generation Strategy

**Pattern:** Slug + Long ID (IdGen) + Crockford Base32 encoding

Use this skill when creating entities or implementing features that need IDs.

## When to Use This Skill

- Creating new domain entities (Skill, Course, User, etc.)
- Implementing features that generate IDs
- Encoding IDs for public APIs or URLs
- Understanding the ID strategy and patterns

## ID Strategy Overview

**Three-part strategy:**

1. **DB primary keys**: Long IDs generated by IdGen (distributed, time-ordered)
2. **Public APIs**: Crockford Base32 encoded IDs (URL-safe, compact, ~13 characters)
3. **Content URLs**: Slug for human-readable URLs (where applicable)

**Benefits:**

- **No DB roundtrip** — IDs are generated in the application
- **Time-ordered** — natural chronological sorting
- **Distributed** — no coordination between servers required
- **Compact encoding** — 13 characters instead of 19 digits
- **URL-safe** — Crockford Base32 in URLs
- **Human-readable** — slug for content (skills, courses, blog)

## Quick Reference

### Entity with ID

```csharp
// Domain/Skills/Skill.cs
public class Skill
{
    public long Id { get; set; }              // Raw long ID (primary key)
    public string Slug { get; set; }          // "test-driven-development"
    public string Title { get; set; }
    public long AuthorId { get; set; }        // FK to users
}
```

### Generating IDs in a Handler

```csharp
public class RequestHandler : IRequestHandler<Request, Response>
{
    private readonly AppDbContext _db;
    private readonly IdFactory _idFactory;

    public RequestHandler(AppDbContext db, IdFactory idFactory)
    {
        _db = db;
        _idFactory = idFactory;
    }

    public async Task<Response> Handle(Request request, CancellationToken ct)
    {
        var skill = new Skill
        {
            Id = _idFactory.Create(),
            Slug = request.Slug,
            Title = request.Title
        };

        await _db.Skills.AddAsync(skill, ct);
        await _db.SaveChangesAsync(ct);

        return new Response(skill.Id);
    }
}
```

### Encoding IDs for a Response

```csharp
public record SkillResponse(string Id, string Slug, string Title)
{
    public static SkillResponse FromEntity(Skill skill)
    {
        return new SkillResponse(
            IdEncoding.Encode(skill.Id),
            skill.Slug,
            skill.Title);
    }
}
```

### Public API Response

```json
{
  "id": "3g5w7x9y1z2a",
  "slug": "test-driven-development",
  "title": "Test-Driven Development"

```

### URL Patterns

```text
GET /api/skills/test-driven-development      // Slug preferred for content
GET /api/users/3g5w7x9y1z2a                  // Encoded ID for internal entities
```

## Key Rules

1. **DB**: Always use raw `long` IDs for primary and foreign keys
2. **Public APIs**: Always encode IDs in JSON responses via `IdEncoding.Encode()`
3. **URLs**: Prefer slug for content entities, encoded IDs for internal entities
4. **Never decode in requests**: Use slug or raw ID, not encoded IDs
5. **Inject IdFactory**: Use DI to inject `IdFactory` into handlers
6. **Generate before saving**: Call `_idFactory.Create()` before adding to DbContext

## Configuration

```json
{
  "IdGen": {
    "GeneratorId": 0,
    "Epoch": "2024-01-01T00:00:00Z"
  }
}
```

**GeneratorId:** Unique per server instance (0-255)
**Epoch:** Start date for timestamp calculation (recommended 2024-01-01)
